Fully decentralized for Defi and Web3 apps

An economic incentive infrastructure platform for blockchains to make their data accessible reliably and useful to end-users, developers, and web3 applications.

Why Massbit?

Massbit solves current blocking issues of blockchain in order to make web3 app fully decentralized.

Decentralized blockchain gateway

Infura was outage once and it caused exchanges like Binance, Bithumb unable to send transactions to withdraw funds. A decentralized gateway run by the community like Bitcoin, Ethereum is needed to make sure the transactions are successfully submitted to a specific blockchain network.

Decentralized backend to generate data insight or to do any generic computation for web3 app

Every web3 app currently needs a centralized service to mine blockchain data to provide the data insight to the app. It is not only an unreliable service as it can get outaged anytime as of centralization but also causes the redundancy of building such services from many apps. A decentralized backend protocol run by the community to organize specific data insight for blockchains is necessary for the web3 app.

Continuous fraud monitoring

On top of the decentralized gateway, we can monitor blockchain and p2p network for frauds such as eclipse 0-confirmation attacks, eclipse attacks. And the service provides end-users, developers with protection and confidence in doing crypto transactions.

Decentralized IPFS access

Similar to decentralized blockchain gateway, I'd like to build a network of IPFS nodes and provide a scalable IPFS service on top of the network based on economic incentives.

Massbit Platform

A platform to incentivize the community to run workers for doing the same tasks. That is to avoid the single point of failure and the incentives are to make sure they all behave well.

Participants have to stake MAB tokens and do the work properly to get rewards. Otherwise, they will get slashed and lose their staked MAB tokens

Our Products

Product 1: Economic incentive platform for blockchain nodes running with MassBit protocol

This gives consumers, centralized app, or web3 app, and smart contract the confidence to access reliable and integrity data source Other oracle projects can talk to our pool of reliable nodes to send and receive transactions/events.

The contributor nodes have to stake MAB tokens to run with MassBit platform. The nodes will not only receive their own node rewards/transaction fees but also receive MAB tokens for their contribution. The nodes will have their data accessible to data mining nodes of MassBit protocol (describe later) The nodes will have more chances to receives first and relay them other nodes and get more transaction fees reward.

Product 2: Data insight analytics

Blockchain data insight to web3 app via graphql or Rest API
Oracle access via smart contract
Or provide access to other oracle chains (chainlink, Band Protocol, etc..)
As we can access multiple source data of reliable data,by economically incentives, we can let data run MassBit miners with specific set of tasks to organize and mine the data to have them useful and accessible to Web3 app or consumer via a gateway which routes the requests to the miners based on their stakes.
The bad or non-reliable nodes will be slashed and lose their stakes.
